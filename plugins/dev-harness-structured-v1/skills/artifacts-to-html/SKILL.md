# artifacts-to-html — 마크다운 산출물 → 자족형 HTML 내보내기

이 플러그인(`dev-harness-structured-v1`)의 모든 문서 산출물은 마크다운(`.md`)으로 생성된다.
**유일한 예외**는 통합 진행 현황판 `docs/pm/progress.html`(이미 HTML, 단일 진입점).
이 스킬은 **요청 시** 마크다운 산출물을 **현재 방식의 자족형 HTML**(다크 테마·인라인 CSS·SVG·카드/배지)로
빠르게 변환해, **같은 디렉터리에 같은 이름의 `.html`** 로 내보낸다(`docs/PRD.md` → `docs/PRD.html`).

> 마크다운이 **원천(source of truth)** 이다. HTML은 그 원천에서 만든 **열람·공유용 스냅샷**이다 —
> 내용을 고치려면 `.md`를 고치고 다시 내보낸다. HTML을 직접 편집하지 않는다(다음 내보내기에서 덮인다).

## 언제 쓰나

- 사용자가 "HTML로 만들어/변환/내보내줘", "HTML화", "공유용 HTML" 등을 요청할 때.
- 대상이 특정 문서(예: "PRD만"), 특정 유형(예: "검증 보고 전부"), 또는 전체일 수 있다 — 아래 "대상 선택" 참조.
- 평상시 산출(기획·설계·검증·구현)에는 쓰지 않는다. 그건 마크다운으로 가볍게 둔다.

## 대상 선택

`Glob`로 `docs/**/*.md`를 훑어 변환 대상을 정한다. 사용자가 범위를 지정하면 그에 맞춘다.

| 요청 | 대상 |
|------|------|
| "전체 / 다" | `docs/**/*.md` 전부 |
| "기획" | `docs/PRD.md` · `docs/FeatureSpec.md` · `docs/UserStories.md` |
| "설계" | `docs/architecture/architecture.md` · `docs/architecture/adr.md` |
| "검증 보고 / 리뷰" | `docs/reviews/*.md` |
| "로드맵 / 추적성 / 변경이력" | 해당 단일 파일 |
| 특정 파일명 | 그 파일만 |

- **제외(항상)**: `docs/pm/progress.html`(이미 HTML — 재생성하지 않는다). `docs/Templates/`의 입력 템플릿,
  `docs/Planning/`·`docs/Design/`·`docs/Development/`의 **입력 기준 문서**(산출물이 아니다)는 변환하지 않는다.
- `docs/api-contract.md`는 기본 포함하되, "코드 대조용이라 마크다운으로 두는 게 낫다"는 점을 한 줄 안내한다(사용자가 원하면 변환).

## 출력 규칙

- **같은 경로·같은 이름**의 `.html`로 쓴다: `docs/PRD.md` → `docs/PRD.html`, `docs/reviews/qa_s7_1.md` → `docs/reviews/qa_s7_1.html`.
- 기존 `.html`이 있으면 덮어쓴다(스냅샷 갱신). 원천 `.md`는 절대 지우지 않는다.
- 변환 후 생성·갱신된 HTML 경로 목록을 사용자에게 보고한다.

## 스타일 — 자족형 HTML (현재 방식 유지)

산출 HTML은 **외부 의존 없는 단일 파일**(인라인 CSS, 인라인 SVG)로 만든다. 스타일 결정 순서:

1. **사용자 템플릿 우선** — 프로젝트에 `docs/Templates/_base.html`(또는 해당 유형 `*.html`)이 있으면 그 구조·팔레트를 따른다.
2. **없으면 플러그인 베이스** — 동봉 [`templates/artifact-base.html`](../../templates/artifact-base.html)의 `:root` CSS
   변수(`--bg`·`--panel`·`--ink`·`--muted`·`--accent`·`--read`·`--warn`·`--bad` …)를 **색상 팔레트의 단일 출처**로 삼는다.
   다크 테마·시스템 폰트(`"Segoe UI","Malgun Gothic"`)·헤더(h1+요약+메타)+카드/표 레이아웃을 그대로 쓴다.

> `templates/artifact-base.html`을 **그 문서의 골격으로 복사**한 뒤 본문만 채우는 방식이 가장 빠르고 일관적이다.

## 마크다운 → HTML 변환 규칙 (요소 매핑)

원천 마크다운의 각 요소를 베이스 템플릿의 시각 언어로 매핑한다. **마크다운의 의미를 그대로 보존**하되 표현을 격상한다.

| 마크다운 요소 | HTML 표현 |
|---------------|-----------|
| `# 제목` | `<header>`의 `<h1>` + 바로 아래 한 줄 요약(`.sub`) + 메타 줄(`.meta`: 유형·작성일·단계/회차) |
| `## 섹션` / `### 소제목` | `<h2>`(좌측 강조 바) / `<h3>` |
| 목차(앵커 링크) | 상단 목차 블록(`.panel`) + 각 `<h2>`에 `id` 부여 |
| 본문 문단·목록 | `.panel` 안의 `<p>`/`<ul>` |
| 마크다운 표 | `<table>`(베이스의 표 스타일 — 둥근 모서리·헤더 배경) |
| ```` ```mermaid ```` 코드블록 | **인라인 `<svg>`로 재작도**(아래 "다이어그램" 규칙). `.diagram` 컨테이너 + `.legend` 범례 |
| 텍스트 배지 `**PASS**`/`**FAIL**`/`P1`/`[치명]` 등 | **색상 배지**(`.tag`/스코어카드) — PASS=accent, FAIL/치명=bad, 경고=warn, P2/중간=warn, 참조=read |
| 코드 ``` `inline` ``` / 코드블록 | `<code>` / `<pre><code>`(베이스 코드 스타일) |
| 정량 요약(건수·통과율 등) | **카드/스코어카드**(`.panel`을 가로로 묶어 KPI 카드처럼) |
| 각주·가정·리스크·링크 | `<footer>` |

### 다이어그램 (mermaid → 인라인 SVG)

- 각 ```` ```mermaid ```` 블록을 베이스 템플릿의 SVG 컨벤션(노드=둥근 사각형, 화살표 마커 `#arr`,
  입력=`--read`/산출=`--accent`/흐름=`--gray`)으로 **인라인 `<svg>`** 로 다시 그리고 **범례(`.legend`)** 를 붙인다.
  결과는 외부 의존 없는 자족형이라 **현재 방식과 동일한 시각 품질**을 유지한다.
- 손으로 SVG화하기에 너무 복잡한 다이어그램은 무리해서 그리지 말고, 원천 mermaid 소스를 `.panel`의
  `<pre>`로 그대로 보존한다(깨진 그림보다 읽히는 소스가 낫다). 어느 쪽이든 **원천 mermaid를 HTML 주석으로 함께 남겨** 추후 재작도에 대비한다.

### 보존 원칙

- 표 컬럼·요구 ID(`R-`/`F-`/`N-`)·검증 회차·판정 결과 등 **내용은 1:1 보존**한다. 변환은 표현 격상이지 재작성이 아니다.
- 문서의 **표준 섹션 순서**(생성 에이전트 정의)를 그대로 유지한다.
- 본문은 한국어 그대로.

## 절차

1. **대상 수집**: `Glob`로 `docs/**/*.md`를 찾고, 사용자 요청 범위로 좁힌다. 제외 대상(progress.html·입력 폴더)을 뺀다.
2. **베이스 확정**: `docs/Templates/_base.html` 존재 여부를 확인해 스타일 출처를 정한다(없으면 `templates/artifact-base.html`).
3. **변환**: 각 `.md`를 `Read`로 읽고, 위 매핑 규칙으로 자족형 HTML을 만들어 **같은 경로의 `.html`**로 `Write`한다.
   - 문서가 많으면 유형별로 나눠 진행하되, 모든 대상이 끝날 때까지 누락 없이 처리한다.
   - 표·다이어그램이 많은 큰 문서(예: PRD·architecture)는 섹션 단위로 꼼꼼히 매핑한다.
4. **검증(가벼움)**: 각 산출 HTML이 ① 단일 파일로 열리는지(외부 링크 의존 없음) ② `<h1>`/섹션/표/배지가 빠짐없이 들어갔는지
   ③ 원천 `.md`의 섹션 수와 대응되는지 눈으로 확인한다. mermaid를 SVG화한 경우 노드·화살표가 원본 흐름과 일치하는지 본다.
5. **보고**: 생성·갱신한 HTML 경로 목록과, (있으면) SVG화하지 못해 소스로 보존한 다이어그램을 한 줄로 알린다.

## 안티패턴 (하지 말 것)

- **원천 `.md` 삭제·수정** — HTML은 스냅샷일 뿐, 원천은 마크다운이다. `.md`는 보존한다.
- **`docs/pm/progress.html` 재생성** — 현황판은 이미 HTML이고 PM이 직접 관리한다. 이 스킬의 대상이 아니다.
- **외부 CDN·스크립트 링크 삽입** — 자족형 원칙을 깬다. mermaid.js 등을 링크하지 말고 인라인 SVG로 그린다.
- **입력 기준 폴더(`Templates/`·`Planning/`·`Design/`·`Development/`) 변환** — 산출물이 아니다.
- **내용 재작성** — 변환은 표현 격상이다. 표·ID·판정·섹션 순서를 임의로 바꾸지 않는다.

## 테스트 시나리오

**전체 변환:** 사용자 "산출물 전부 HTML로 만들어줘"
→ `Glob docs/**/*.md` → progress.html·입력 폴더 제외 → 각 파일을 `templates/artifact-base.html` 골격으로 변환,
표→`<table>`·배지→`.tag`·mermaid→인라인 `<svg>` → 같은 경로에 `.html` 작성 → 생성 목록 보고.

**부분 변환:** 사용자 "검증 보고만 HTML로"
→ `docs/reviews/*.md`만 대상 → 각 보고를 판정 배지·이슈 표 중심으로 HTML화 → `docs/reviews/*.html` 작성 → 보고.
